# NetWise 操作文档

本文档说明 **NetWise（netwise-mobile）** 的安装、运行与日常操作。当前版本以 **Expo + React Native** 构建，面向个人金融截图的本地 OCR、解析、汇总与趋势展示。

---

## 1. 产品定位与能力概览

| 能力 | 说明 |
|------|------|
| 截图导入 | 相册多选（最多约 6 张）或从文件选择（模拟器推荐） |
| OCR | 设备本地 **Google ML Kit** 文本识别，不上传截图到外部 OCR 服务 |
| 页面识别 | 内置招行/支付宝/微信等模板；支持自定义 OCR 规则与自定义「识别模块」 |
| 解析与编辑 | 自动解析金额与资产分类，可在确认前手动增删改 |
| 持久化 | 确认后写入本地 **加密** 快照文件；按自然日、图片 hash 去重 |
| 趋势图 | 总资产趋势；支付宝 / 招行 / 微信分平台趋势；自定义模块趋势；可按资产类筛选 |
| 安全 | 6 位数字口令；可选生物识别解锁；快照内资产数据加密存储 |

## 2. 环境准备（开发者）

### 2.1 必备软件

- **Node.js** LTS（建议 20+）
- **Git**
- 真机调试：**Expo Dev Client** 或按项目脚本打的调试包（本项目含原生 ML Kit，需原生构建，不能仅靠 Expo Go 扫码即用，以实际 `package.json` 脚本为准）
- Android：`Android SDK`、`adb`；项目脚本中示例路径为 `D:\Environment\Android\Sdk`（可按本机修改）

### 2.2 安装依赖

在项目根目录执行：

```powershell
npm install
```

PowerShell 下若串联命令，请用分号 `;` 而非 `&&`（视系统版本而定）。

### 2.3 启动开发服务

常用命令（以 `package.json` 为准）：

| 命令 | 用途 |
|------|------|
| `npm run start` | 清理 8081、adb reverse、离线启动 dev client 等（Windows 定制脚本） |
| `npm run start:clean` | 清理端口后以 LAN 方式启动 |
| `npm run start:online` | 在线 `expo start --dev-client` |
| `npm run android` | `expo run:android` 编译/安装到设备 |
| `npm run typecheck` | TypeScript 检查，改解析规则后建议执行 |

首次接入或升级 **本地 OCR 原生模块** 后，需重新安装调试包或重打 APK；仅靠热更新无法把新原生模块带入已安装 App。

### 2.4 类型检查

```powershell
npm run typecheck
```

## 3. 终端用户操作说明

### 3.1 首次启动：口令与解锁

1. 首次打开需设置 **6 位数字口令**。
2. 之后每次进入主界面需 **输入口令** 解锁；若设备支持并已录入生物特征，可按提示使用 **生物识别**（可在流程中按应用逻辑开启/关闭偏好，以界面为准）。
3. 本地资产快照为 **加密存储**；不解锁无法查看汇总与趋势。

### 3.2 导入截图

1. 在导入区域选择图片来源：**相册** 或 **文件**（模拟器上文件方式更稳定）。
2. 可选 **多张**（上限以界面提示为准，当前实现约 **6 张**）；可点缩略图全屏预览，角标可移除单张。
3. 授予 **相册/文件** 权限；若拒绝，按系统设置开启后重试。

### 3.3 OCR 与解析结果

1. 识别完成后会显示 **页面类型**（如招行财产页、支付宝理财页等）或 **未识别/自定义模块** 等状态。
2. **解析结果** 区域可：
   - 修改 **金额名称**、**金额**、**资产分类**（现金 / 基金 / 保险 / 股票 / 理财）；
   - **删除行**、**手动添加一行**；
   - 展开 **查看 OCR 原文** 核对（若存在）。
3. 确认无误后点击 **「确认并记录」**（或等价文案）写入当日快照。

**注意：** 确认后通常会 **清除当前导入图片预览与内存引用**；原始截图 **不会** 作为图片文件写入资产历史文件。

### 3.4 去重规则（当日）

- 以 **图片内容 hash** 为键。
- **同一自然日** 内，同一 hash **只记录一次**；重复导入同一张图当日不会重复落库。

### 3.5 首页汇总与趋势图

- **目前为止总资产** 及按类的汇总，与已确认快照一致。
- **资金趋势**：主图支持 **全部** 或按 **资产类** 筛选；「全部」时可能叠加多条分项折线（视数据而定）。
- **分平台**：支付宝、招商银行、微信各自趋势卡片，筛选方式与主图类似。
- **自定义识别模块**：在设置中配置后，命中关键词的导入会计入对应模块折线；可通过 **× / +** 控制是否在首页展示（**不删除** 历史数据）。

点击图表可查看 **单日** 各线金额（以当前实现为准）。

### 3.6 设置

进入 **设置** 后常见项包括：

- **隐私与数据说明**（界面内「i」浮层）：加密、hash 去重、确认后清预览等。
- **自定义识别模块**：新建向导（选图 → OCR → 匹配词「或」关系 → 展示名称）；点击模块名进入 **配置/删除**。
- **自定义 OCR 规则**：规则存于本机 `netwise-ocr-custom-rules.json`；**原文** 为锚点子串（OCR 去空白后须包含）；金额取该锚点 **之后第一个数字**；可选 **限定页面** 或自定义模块作用域以减少误匹配。
- **数据清理**：
  - 清除 **今日** 已确认快照；
  - **清空全部导入**（仅快照，**不删** 自定义模块与 OCR 规则）；
  - **测试数据**：写入模拟曲线（**勿在日常账目中误触**）；可 **清除测试数据** 或依赖「清空全部导入」。
- **调试**（若开启「测试数据」或调试开关）：可查看 **已存快照 JSON**、折线数据结构等，用于开发联调。

## 4. 本地数据文件（了解即可）

| 文件（应用文档目录下） | 内容 |
|------------------------|------|
| `netwise-asset-history.json` | 导入快照（**AES 加密** 后的 JSON） |
| `netwise-ocr-custom-rules.json` | 自定义 OCR 规则（明文 JSON） |
| `netwise-custom-recognition-modules.json` | 自定义模块与隐藏展示 id（明文 JSON） |

口令哈希与加密密钥等保存在 **SecureStore** 中，勿手动篡改。

## 6. 常见问题

| 现象 | 处理建议 |
|------|-----------|
| 相册选图不稳定（模拟器） | 使用「从文件导入」；或用 `adb push` 推到 `/sdcard/Pictures/` 再选 |
| 提示本地 OCR 模块未编入 | 重新 `expo run:android` 或重装含原生模块的 APK |
| 8081 端口占用 | 使用 `npm run start:clean` 或 `npm run stop:metro` |
| 改代码未生效 | 在 Metro 终端按 `r` 刷新 |
| 解析错误 | 展开 OCR 原文；在 `src/parsers/templates.ts` 调整规则后 `npm run typecheck` |
